Development of Shortest Path Applications for Public Transport Timetables

Dear Mr Prasad,

Hello,

Mr Prasad I have come across a number of your recent papers on a fast and flexible implementation of the Connection Scan Algorithm (CSA).

Mr Prasad I would like to draw upon your services to assist me with the development of:

[a] Fast parallel execution C# command line/console public transport timetable shortest path routing application.

The command line/console application is expected to support the specification of all input data/parameters and output data/results filenames (e.g. origin-destination results files/matrices of journey times/distances/costs, shortest path routes) specified within the command line and/or within an associated initialisation/setup text/xml file.

and;

[b] Fast parallel execution, lightweight, maintainable and scalable REST service supporting public transport timetable shortest path routing.

These two applications will need to support:

[1] Fast and efficient construction and storage of i) one to one, ii) one to many, iii) many to one and iv) many to many shortest path trees, shortest path results files/matrices and associated trees.

[2] Usage on country wide public transport timetable datasets.

[3] Public transport routing options encompassing specification of:

a) Day of Week: Monday, Tuesday, Wednesday, Thursday, Friday, Saturday, Sunday, Public Holiday [multiple selection]. If Date is specified then Day of Week is omitted in shortest path computations.
b) Date (dd/mm/yyyy). If [Day of Week] is specified then [Date] is omitted in shortest path computations.
c) Mode of Travel: Bus, Coach, Ferry, Light Rail, Metro, Plane, Rail, Tram [multiple selection].
d) Direction of Travel: Inbound, Outbound [single selection].
e) Routing Criteria: Depart After (hh:mm) & Arrive Before (hh:mm).
f) Maximum No. of Interchanges.
g) Maximum Interchange Time (hh:mm).
h) Interchange Penalty (hh:mm).
i) Maximum Access/Egress Walk Distance By Stop Type (km).
j) Maximum Interchange Walk Distance By Stop Type (km).
k) Walk Network: Highway Network, Cycle Network, Footpath Network, Walk Crows Flight Path [multiple selection].
l) Walk Crows Flight Path Adjustment Factor.

[4] The development of a PostgresSQL data importer for the General Transit Feed Specification (GTFS) data which will provide an input PostgresSQL database containing public transport timetable data to be used in building of public transport journey time shortest paths and public transport journey time results files/matrices and associated shortest path trees/routes.

https://developers.google.com/transit/gtfs/reference?hl=en

[5] Mr Prasad I will take your advice on whether to utilise your fast and flexible implementation of the Connection Scan Algorithm (CSA) or the Delling et al RAPTOR Algorithm (RA) (https://www.microsoft.com/en-us/research/publication/round-based-public-transit-routing/) within the command line/console application and REST service.

Mr Prasad my principle requirement is ostensibly a very fast and efficient algorithm and application which supports the aforementioned criteria.

Mr Prasad if this project would be of interest to you please let me know so that we can schedule a Skype call to discuss further.

Many thanks in advance,

Sincerely yours,

Olu.

Olu Ashiru.
